# 分部估值 (SOTP) 方法论

## 何时使用 SOTP

**触发条件：**
- 公司在年报中报告 2+ 个经营分部
- 各分部处于不同行业（如科技+零售、媒体+主题公园）
- 某分部增长明显更快或更有价值
- SOTP 分析显示 >20% 上行（显著集团折价）
- 12-24 个月内有合理催化剂：维权投资者、战略评估、拆分传闻

**不要强行使用 SOTP：**
- 分部间运营深度整合（拆分会破坏协同）
- 分部披露过于粗糙无法独立建模
- 管理层不可能拆分且无外部压力

## 工作流

### 步骤 1: 提取分部财务数据

从最新年报/季报分部披露中提取每个分部的：
- 营收
- 营业利润 (EBIT)
- EBITDA（如披露，否则 EBIT + 分配的 D&A）
- 营收同比增长
- 营业利润率

跨分部抵销和未分配公司费用单独记录。

### 步骤 2: 识别纯业务同业

每个分部找 3-5 家同行业上市纯业务同业：

| 分部类型 | 纯业务同业示例 |
|---|---|
| 云基础设施 | MSFT (Azure), AMZN (AWS), GOOGL (GCP) |
| 数字广告 | META, GOOGL, TTD, PINS |
| 流媒体 | NFLX, DIS (DTC), WBD |
| 半导体（设计） | NVDA, AMD, AVGO, MRVL |
| 半导体（代工） | TSM, INTC (IFS), GFS |
| 零售（实体） | WMT, TGT, COST, HD |
| 汽车（传统） | F, GM, STLA |
| 保险（财产险） | TRV, CB, ALL, PGR |
| 公用事业（受管制） | DUK, SO, AEP |

记录同业中位数 EV/EBITDA、EV/Revenue。

### 步骤 3: 应用倍数

```
分部 EV_i = 分部 EBITDA_i × 同业中位数 EV/EBITDA_i
```

默认用 EV/EBITDA。高成长或未盈利分部用 EV/Revenue。

### 步骤 4: 公司级调整

```
分部 EV 合计
− 未分配公司成本 (取营收的 2-5%，或按 8x 持续成本折现)
− 少数股东权益
− 总债务
− 优先股
− 养老金缺口
+ 现金及等价物
+ 非经营资产 (多余房产、投资、税损转回)
= 股权价值

隐含价格 = 股权价值 / 稀释后流通股
```

### 步骤 5: 计算集团折价

```
折价% = (SOTP价格 − 市场价格) / SOTP价格 × 100
```

| 折价幅度 | 含义 |
|---|---|
| >30% | 引人注目；大概率可操作 |
| 20-30% | 显著；需要催化剂 |
| 10-20% | 窄幅；需催化剂+技术面配合 |
| <10% | 无机会 |

### 步骤 6: 识别催化剂

集团折价可以无限期存在。至少需要以下一个催化剂：
- 维权投资者提交 13D 推动拆分
- 管理层公开讨论"战略选项"或"组合优化"
- 传闻或公告拆分/剥离
- CEO 更换（新 CEO 常倾向简化）
- 同业交易凸显估值差距
- 董事会改组（维权投资者提名董事）

## 边界情况

| 问题 | 处理方式 |
|---|---|
| 共享成本分配不一致 | 阅读分部脚注；任意分配的需重新计算 |
| 协同破坏 | 对运营耦合的分部 EV 扣除 5-15% |
| 拆分税务泄漏 | 实现价值的 10-20% 作为税务成本 |
| 分部中有少数股东权益 | 分部 EV × 母公司持股比例 |
| 隐藏负债 (环保、诉讼) | 扣除估计 NPV |
| 无催化剂的持续折价 | 不建议 — 催化剂出现前是"沉没资金" |

## yfinance 数据限制

yfinance **不提供**分部级财务数据。SOTP 分析需要：
- 用户手动提供分部数据，或
- Agent 从 SEC/港交所原始文件解析

如无分部数据，跳过 SOTP，仅用 DCF + 相对估值。在输出中说明跳过原因。
