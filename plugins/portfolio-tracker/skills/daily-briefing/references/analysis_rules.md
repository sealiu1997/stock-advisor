# 分析规则引擎

## 1. 主线检测

每次播报必须先确定 1-2 个市场主线。主线从以下 6 类驱动因素中选取。

### 驱动因素检测标准

| 驱动类型 | 检测信号 | 典型主线示例 |
|---------|---------|------------|
| 宏观数据 | 当日或前一交易日有重大数据发布（CPI/NFP/GDP/PCE/PMI），且实际值偏离预期 | "CPI 低于预期 → 降息预期升温" |
| 利率驱动 | 2Y/5Y/10Y 美债收益率单日变动 > 5bp，或 DXY 单日变动 > 0.5% | "长端利率上行 → 成长股承压" |
| 地缘政治 | 一级新闻源报道重大地缘事件（军事冲突、制裁、贸易摩擦升级） | "中东冲突 → 避险交易回归" |
| 商品驱动 | 黄金/原油/铜单日变动 > 1.5%，且有明确事件驱动 | "OPEC 减产 → 油价飙升 → 通胀预期抬头" |
| 行业驱动 | 某 GICS 板块 ETF 单日变动 > 2%，或多只同板块股票同向异动 | "AI 芯片需求超预期 → 半导体链集体上涨" |
| 个股事件 | 持仓或关注列表中有财报/并购/监管等重大事件 | "NVDA 财报超预期 → AI 主线强化" |

### 主线选取规则

1. **优先级**：宏观数据 > 利率驱动 > 地缘政治 > 商品驱动 > 行业驱动 > 个股事件
2. **必须有证据**：主线必须至少有 1 条一级新闻源或官方数据支撑
3. **最多 2 条**：避免散。如果当日确实有 2 个独立强驱动（如 CPI + 中东冲突），可以写 2 条
4. **找不到就说找不到**：如果当日无明确驱动因素，主线写"市场缺乏明确方向，窄幅震荡"
5. **连续性检查**：读取 PKS 中最近的活跃叙事，判断今天是"延续"还是"转折"

## 2. 标的分析触发规则

确定主线后，对持仓和关注列表中的标的进行分层：

### A 层：必须详细分析

满足**任一**条件：
- 单日涨跌幅超过 `must_analyze` 阈值（config/briefing.json）
- 当日有重大新闻（财报、并购、监管、诉讼等）
- 与当日主线**强相关**（板块直接受影响、被主线新闻点名）
- 成交量/换手率异常（> 近 20 日均值的 2 倍）

### B 层：简要提及

满足**任一**条件：
- 单日涨跌幅超过 `brief_mention` 阈值但未达 `must_analyze`
- 与当日主线**弱相关**（同板块但非直接影响）
- PKS 中有活跃叙事与该标的相关

输出：一句话，说明方向和可能原因。

### C 层：跳过

同时满足**所有**条件：
- 涨跌幅低于 `brief_mention` 阈值
- 无重大新闻
- 与当日主线无关
- 无成交量异常

输出：不单独提及。末尾统一用 `output.skip_message` 一笔带过。

## 3. 归因证据链标准

每条标的分析必须遵循 **因子 → 传导 → 个股** 三层链条。

### 模板

```
**一级因子**：发生了什么（事件/数据/新闻）
**传导机制**：为什么影响该类资产（逻辑链条）
**个股映射**：为什么具体影响到这只标的（业务关联、敞口大小）
```

### 示例

```
**一级因子**：美国 6 月 CPI 3.1%，低于预期 3.2%
**传导机制**：市场上修 H2 降息预期 → 10Y 收益率下行 8bp → 成长股估值压力缓解
**个股映射**：GOOG 作为高估值科技股，对贴现率变化敏感；纳指涨 1.2% 提供β支撑
```

### 禁止的归因模式

以下模式说明证据不足，**不准输出**：

| 模式 | 问题 |
|------|------|
| "可能受益于市场情绪改善" | 没有说是什么情绪、什么驱动 |
| "受行业利好消息推动" | 没有说是什么利好 |
| "涨幅主要受大盘带动" | 没有解释大盘为什么涨 |
| "技术面超卖反弹" | SKILL 不做技术分析，不是我们的能力圈 |
| "长期看好该公司发展前景" | 与当日行情无关 |

**如果确实无法找到可靠归因，应该写：**
> "该标的今日波动 X%，但未找到明确的新闻或事件驱动。无充分证据进行归因分析。"

## 4. 证据置信度

每条分析附带置信度标注：

| 等级 | 标准 | 输出规则 |
|------|------|---------|
| **高** | 有一级新闻源/官方数据直接支撑 + 传导逻辑清晰 | 必须输出 |
| **中** | 市场逻辑合理 + 间接证据（二级源、同板块联动） | 可输出，标注"间接推断" |
| **低** | 仅有价格联动，无新闻/事件支撑 | **不输出** |

`config/briefing.json` 中 `output.confidence_threshold` 控制最低输出等级，默认 `medium`。

## 5. 跨市场联动分析规则

### 美股 → 港股

美股收盘后的港股早报必须包含：
- 隔夜美股主线是什么
- 该主线对港股的传导路径（如：纳指涨 → 港股科技股高开预期）
- 注意时差：美股盘后的 after-hours 消息也要覆盖

### 港股 → 美股

港股收盘后的美股前瞻可提及：
- A/H 溢价变化
- 南向/北向资金（如果可获取）
- 中国政策信号对在美中概股的影响

### 商品 ↔ 宏观

黄金、原油、BTC 的分析必须回到宏观框架：
- 黄金：实际利率 (10Y - CPI)、美元强弱、避险需求
- 原油：供给 (OPEC)、需求 (PMI/GDP)、地缘溢价
- BTC：风险偏好、美元流动性、监管

## 6. 报告结构

### 标准 5 段式

```markdown
## {市场}播报 | {日期}

### 1. 今日主线
> 一句话定调。必须有证据源标注。

### 2. 宏观关键信号
| 指标 | 当前值 | 变动 | 信号 |
（只列与今日主线最相关的 3-5 项，不全念一遍）

### 3. 重要新闻 / 事件
- 来源 + 摘要（2-4 条，一级源优先）

### 4. 持仓分析
（A 层：详细分析，每个走因子→传导→映射链条）
（B 层：一句话简述）
（C 层：统一跳过语）

### 5. 下一时段关注
- 关注变量（如明日 CPI 发布、FOMC 纪要、某股财报）
- 来自 PKS 中活跃叙事的风险因素
```

### 紧凑模式（定时 cron 用）

当由 cron 触发时，压缩为更紧凑的格式：
- 主线 1 句
- 关键数据 1 个表格
- 异动持仓 1-3 条（A 层）
- 下一时段关注 1-2 条
- 总长度控制在 500 字以内
